How people judge the size of shapes in rectangles and treemaps



We will follow the following steps to design the study assuming we have stimuli (static images of instances) for rectangles (download stimuli) and stimuli for treemaps (download stimuli):
  • Creating viewers: First, we will include HTML files that will represent the visualizations for rectangle and treemap and will be used to display the stimuli.

    Then use the add new viewer form to create a viewer directory for Rectangles and upload the rectangle stimuli files and the HTML file (rectangle.zip).

    Similarly, create a viewer directory for Treemap and upload the treemap stimuli files and HTML file(treemap.zip).
  • Creating tasks: Use the new task form to create the following 2 tasks. Task#1: Which of the two rectangles (marked A or B) is smaller?. Task#2: Estimate the percentage of the smaller rectangle to the larger one.

    These tasks can be created from scratch as demonstrated here (task1, task2)

    Or by uploading the following task xml files for task1 and task2 using the upload link on the new task form

    Note: These tasks require an input of the type "stimuli", as such, the HTML files for the visualizations (i.e. the rectangle and treemap visualizations) need to include a setStimuli interface method that VisUnit will use to pass the input of each task instance to the visualization. For example, the interface method for the rectangle can be as follows: 
    
 function setStimuli(name) {
      document.getElementById("stimuliImage").setAttribute("src", name);
 }
  • Creating task instances: Use the upload link on the new task Instances form to upload the following taskInstance xml files for the rectangle and treemap viewer files as demonstrated in images (rect_t1.xml | pic) , (rect_t2.xml | pic) ,(treemap_t1.xml | pic) , (treemap_t2.xml | pic)
